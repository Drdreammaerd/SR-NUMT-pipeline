# Data Dictionary — SMaHT NUMT Pipeline

This data dictionary explains the output formats and column definitions across all three stages of the NUMT pipeline:
1. **Stage 0: Discovery** (dinumt raw calls)
2. **Stage 1: Validation** (Single-donor cross-replicate validation)
3. **Stage 2: Catalog** (Cross-donor population integration)

---

## Stage 0: Discovery (dinumt short-read NUMT callset)

### 1. `output/dinumt/<donor_id>_fullBAM_AllNumts/<tissue>.vcf`

The raw output VCF from `dinumt` for a specific tissue replicate (e.g., `uwsc_BLOO_1.vcf`).

**VCF Columns:**

| Column | Meaning |
|--------|---------|
| `CHROM` | The target nuclear chromosome where the mitochondrial insertion occurred (e.g., `chr1`). |
| `POS` | The estimated 1-based start position of the insertion site on the nuclear chromosome. |
| `ID` | Identifier generated by dinumt. |
| `REF` | The reference allele base at the specified POS. |
| `ALT` | The alternative allele. Explicitly `<INS:MT>`, signifying a Nuclear Mitochondrial Insertion. |
| `QUAL` | The Phred-scaled quality score for the assertion that there is a variant at this position. |
| `FILTER` | `PASS` if the variant passes all quality control checks. Common failures: `q50` (QUAL < 50), `e4` (supported by < 4 read pairs), `d5` (local depth < 5). |
| `INFO` | Structural characteristics (see INFO Fields table below). |
| `FORMAT` | Format keys for sample-level metrics (e.g., `GT:FT:GL:GQ:PL`). |
| `<tissue>` | Sample-level metrics. Tissue naming convention is `<center>_<organ>_<replicate>`. |

**Key INFO Fields:**

| Field | Meaning |
|-------|---------|
| `CIPOS` / `CIEND` | Confidence intervals around POS and END as negative/positive integer offsets. |
| `END` | End coordinate of the SV interval on the nuclear reference. |
| `IMPRECISE` | Flag indicating breakpoint boundaries are approximate. |
| `MSTART` / `MEND` | Coordinates on the mitochondrial genome mapping to the inserted sequence. |
| `MLEN` | Estimated length of the inserted mitochondrial sequence. |
| `SVTYPE` | Always `INS` (Insertion). |

**Key FORMAT Fields:**

| Field | Meaning |
|-------|---------|
| `GT` | Genotype (`0/0`, `0/1`, `1/1`). |
| `FT` | Per-sample genotype filter. |

### 2. `output/metadata/<donor_id>_final.tsv`

The final manifest generated at the end of Stage 0, summarizing the discovery phase.

| Column | Meaning |
|--------|---------|
| `TISSUE` | Tissue ID in the format `<center>_<organ>_<replicate>` (e.g., `uwsc_BLOO_1`). |
| `FULL_BAM_PATH` | Absolute path to the input BAM/CRAM file. |
| `MEAN_INSERT_SIZE` | Mean template length for paired and mapped reads (from chr20 subsample). |
| `INSERT_SIZE_SD` | Standard deviation for template length distribution. |
| `DINUMT_VCF` | Absolute path to the output raw VCF. |
| `RAW_COUNTS` | Number of unfiltered candidate nuclear-mitochondrial discordant read pairs. |

## Stage 1: Single-Donor Validation

Outputs are located in `output/<donor_id>/`.

### 1. `validation/Organ/Union_VCFs/<organ>_union.vcf`

A merged VCF clustering all `dinumt` calls from all replicates of a specific organ (e.g., `HART_union.vcf`).

| Column / Field | Meaning |
|----------------|---------|
| `ID` | Assigned as `UNION_XXXX` (e.g., `UNION_0001`). |
| `INFO: NSRC` | Number of source replicates that contributed to this cluster. |

### 2. `<donor_id>_Presence_Matrix.csv`

The primary output matrix for the donor.

| Column | Meaning |
|--------|---------|
| `<organ_name>` | One column per organ. Values represent Variant Allele Frequency (`VAF%`). `0.0` if not detected/validated. |
| `Coordinates` | Consensus genomic coordinates (e.g., `chr1:733000-733050`). |
| `Mito_Source` | Inserted region from the mitochondrial genome. |
| `NUMT_Class` | Single-donor biological classification (`Germline`, `Mosaicism`, `Somatic`, `Unclassified`). *See NUMT Class Definitions below.* |
| `Best_Confidence` | Highest technical replicability observed across any organ (`HighConf`, `MediumConf`, `LowConf_SingleCenter`, `LowConf_SingleRep`, `Not_Detected`). |
| `Total_Validated_Organs` | Count of organs where NUMT was validated. |
| `Total_Organs` | Total organs sequenced for this donor. |
| `Validated_Organ_List` | Comma-separated list of organs where the NUMT was found. |
| `Missing_Organ_List` | Organs where the NUMT was not found. |

**NUMT Class Definitions:**

| Class | Definition |
|-------|------------|
| `Germline` | Validated in *every* organ sequenced for the donor. |
| `Mosaicism` | Validated in >1 organ, but not all. |
| `Somatic` | Validated in exactly 1 organ. |
| `Unclassified` | No organs met the technical confidence thresholds required to be considered a true biological variant. To pass, a NUMT must meet ALL of the following criteria in at least one organ replicate:<br>1. $\ge$ 4 supporting `Alt` reads<br>2. $\ge$ 1.0% Variant Allele Frequency (`VAF`)<br>3. `Noise_Ratio` $\le$ 5.0 (Noisy reads cannot exceed 5x the true signal)<br>4. `Strand_Ratio` between 0.2 and 0.8 (Requires support from both forward and reverse strands to avoid sequencing bias) |

### 3. `reports/<donor_id>_Replicate_Detail.csv`

Highly detailed table tracking the exact validation metrics for every NUMT in every replicate BAM.

| Column | Meaning |
|--------|---------|
| `NUMT_ID` | Global NUMT ID for this donor. |
| `Tissue` | Replicate ID (e.g., `uwsc_HART_1`). |
| `Alt` | Number of reads supporting the MT insertion. |
| `Ref_Avg` | Average number of reference-supporting reads spanning the breakpoints. |
| `Noisy_Reads` | Reads that map to the locus but don't support REF or ALT clearly. |
| `Total_Depth` | Total read depth at the locus. |
| `Noise_Ratio` | Ratio of noisy reads to signal (Alt + Ref). |
| `VAF%` | Variant Allele Frequency (`Alt / (Alt + Ref_Avg) * 100`). |
| `Strand_Ratio` | Proportion of supporting reads on the forward strand (0 to 1). |
| `Palmer_Alt` | Number of reads supporting the MT insertion in Palmer. |
| `Palmer_Depth` | Total read depth at the locus in Palmer. |
| `Palmer_VAF` | Variant Allele Frequency in Palmer (%). |
| `Palmer_QC` | Palmer Quality Tier (e.g., `NUMT_clean`, `NUMT_partial`). |
| `Evidence_Tier` | Confidence Tier (`Tier 1`, `Tier 2`, `Tier 3`, `LowConf`). *See Evidence Tier Definitions below.* |
| `Status` | `Validated`, `Palmer_Validated`, `LowConf_StrandBias`, or `LowConf`. |
| `Source` | Caller combinations, e.g., `Discovery`, `Rescue`, `Palmer_TSV`, or `Stage2_Rescue, Palmer_TSV`. |

**Evidence Tier Definitions:**

| Tier | Definition |
|------|------------|
| `Tier 1` | **High Confidence:** Found independently by *both* Dinumt (Discovery) and Palmer. |
| `Tier 2` | **Rescue Confidence:** Found only by Palmer initially, but Palmer's coordinates successfully guided Dinumt to rescue the supporting reads from the BAM. |
| `Tier 3` | **Palmer-Only:** Found only by Palmer. Dinumt failed to rescue supporting reads from the BAM. |
| `LowConf` | Failed minimal quality thresholds (e.g., < 4 `Alt` reads, < 1.0% `VAF`, excessive noise, or severe strand bias) in both callers. |

### 4. `reports/<donor_id>_Master_Summary.csv`

A flat list of all unique NUMTs found in the donor.

| Column | Meaning |
|--------|---------|
| `<All Columns>` | Contains the exact same 8 metadata columns found in the Presence Matrix. |

## Stage 2: Population Catalog (Cross-Donor Integration)

Outputs are located in `Outputs/Stage2_Cohort_vX/`.

### 1. `Cross_Donor_Presence_Matrix_<date>.csv`

The master VAF matrix across the entire cohort.

| Column | Meaning |
|--------|---------|
| `POP_NUMT_ID` | Universal ID across the cohort (e.g., `POP_NUMT_0001`). |
| `<donor_id>_<organ>` | Every Organ for every Donor (e.g., `SMHT001_HART`, `SMHT004_HART`). Values are VAF%. `NA` indicates the donor does not have data for that organ. |
| `Consensus_Coordinates` | Merged genomic coordinates covering all donor calls. |
| `Mito_Source` | Inserted region from the mitochondrial genome. |
| `Pop_Classification` | Population-level frequency classification. |

### 2. `catalog/Population_NUMT_Catalog_<date>.csv`

The definitive catalog of all unique NUMTs in the population.

| Column | Meaning |
|--------|---------|
| `POP_NUMT_ID` | Universal ID across the cohort. |
| `Consensus_Coordinates` | Merged genomic coordinates covering all donor calls. |
| `Pop_Classification` | Population-level frequency classification. *See Population Classification Definitions below.* |
| `N_Donors_Germline` | Number of donors where this NUMT is Germline. |
| `N_Donors_Detected` | Number of donors where this NUMT is detected at all. |
| `Donor_Details` | Pipe-separated list of how this NUMT is classified in each donor. |

**Population Classification Definitions:**

| Classification | Definition |
|----------------|------------|
| `Shared_Germline` | Classified as Germline in 100% of donors. |
| `Shared_Mixed` | Detected in 100% of donors, but not classified as Germline in all of them. |
| `Polymorphic_Common` | Classified as Germline in $\ge$50% of the donors (requires N $\ge$ 4 donors total). |
| `Polymorphic_Rare` | Classified as Germline in $\ge$2 donors (but <50%). |
| `Shared_Partial` | Detected in $\ge$2 donors, but Germline in $\le$1 donor. |
| `Donor_Specific_Germline` | Germline in exactly 1 donor (not detected in anyone else). |
| `Donor_Specific_Mosaicism` | Mosaicism in exactly 1 donor (not detected in anyone else). |
| `Donor_Specific_Somatic` | Somatic in exactly 1 donor (not detected in anyone else). |

### 3. `variant_calls/Population_NUMTs_<date>.vcf`

A standard, multi-sample VCF representing the population catalog. Suitable for genome browsers (IGV) and downstream variant tools.

**VCF Columns & FILTER:**

| Field / Column | Meaning |
|----------------|---------|
| `<donor_id>_<organ>` | Sample format (e.g., `SMHT001_BLOO`). |
| `FILTER: PASS` | The variant is classified as `Germline`, `Mosaicism`, or `Somatic` in at least one donor (passes population-level QC). |
| `FILTER: LOWCONF` | The variant failed to pass validation thresholds in all donors (e.g., all are `Unclassified` / `Not_Detected`), indicating it is likely a mapping artifact or noise. |

**INFO Fields:**

| Field | Meaning |
|-------|---------|
| `END` | End position of the NUMT insertion on the nuclear reference. |
| `SVLEN` | Length of the inserted sequence (if calculable). |
| `MITO_SRC` | The mitochondrial coordinates of the inserted sequence. <br><br>**Format 1:** `Start-End` (e.g., `800-1200`) for standard contiguous insertions.<br>**Format 2:** `Junction(Start-16569\|1-End)` for insertions spanning across the mitochondrial origin (D-loop). <br><br>*Note: The old Dinumt VCF used separate MSTART/MEND integer fields, but because population-level coordinates must support circular junctions, they are merged into this string field.* |
| `DISC_SRC` | Caller source combinations across the population (`Dinumt_Only`, `Palmer_Only`, `Both`, `Unknown`). |
| `POP_CLASS` | Overall population classification (e.g., `Shared_Germline`). |
| `NDONORS` | Number of donors in the cohort where this NUMT is detected. |
| `NDONORS_TOTAL` | Total number of donors analyzed in the cohort. |
| `DONOR_CLASS` | Pipe-separated list of the classification in each specific donor. |
| `DONOR_IDS` | Pipe-separated list of the original Stage 1 Donor NUMT IDs. |

**FORMAT Fields (`GT:VAF:DP:ALT:NR:NRR:SR:CF:SRC:P_ALT:P_DP:P_VAF:TIER`):**

| Field | Meaning |
|-------|---------|
| `GT` | Genotype (`0/1` = detected, `0/0` = not detected, `./.` = no data). |
| `VAF` | Mean Variant Allele Frequency. |
| `DP` | Mean Depth. |
| `ALT` | Alt read count. |
| `NR` | Noisy read count. |
| `NRR` | Noise Ratio. |
| `SR` | Strand Ratio. |
| `CF` | Confidence (`HC`=High, `MC`=Medium, `ND`=Not Detected). |
| `SRC` | Source (`D`=Discovery, `R`=Stage2_Rescue, `DP`=Discovery+Palmer, `RP`=Rescue+Palmer, `P`=PalmerOnly). |
| `P_ALT` | Mean Palmer Alt Read Count. |
| `P_DP` | Mean Palmer Depth. |
| `P_VAF` | Mean Palmer VAF (%). |
| `TIER` | Evidence Tier (`Tier 1`, `Tier 2`, `Tier 3`). |

### 4. `variant_calls/Population_NUMTs_<date>.bed`

Standard BED6+ format file for quick visualization in UCSC Genome Browser.

| Column | Meaning |
|--------|---------|
| `<BED Fields>` | Contains coordinate, `POP_NUMT_ID`, population class, and number of donors detected. |

### 5. `details/Cross_Donor_ID_Mapping_<date>.csv`

A traceability matrix to map Population IDs back to the original Stage 1 IDs for each donor.

| Column | Meaning |
|--------|---------|
| `Mapping` | Maps Population ID to Donor ID (e.g., `POP_NUMT_0001` $\rightarrow$ `SMHT001_NUMT_0045` & `SMHT004_NUMT_0012`). |

### 6. `catalog/Donor_Comparison_Summary_<date>.csv`

Summary statistics per donor.

| Column | Meaning |
|--------|---------|
| `<Categories>` | Contains counts for `Total_NUMTs`, `Germline`, `Mosaicism`, `Somatic`, `Shared_With_All_Donors`, and `Unique_To_Donor`. |
